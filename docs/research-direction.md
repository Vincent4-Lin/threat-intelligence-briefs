# Research Direction Fit

## Confirmed Research Direction

AI-assisted cybersecurity investigation and threat intelligence analysis for identifying attack patterns and improving incident response.

## How This Project Supports the Direction

This project supports my research direction by practicing how to turn security incidents and attack scenarios into structured threat intelligence briefs. Each brief organizes the incident context, attack flow, MITRE ATT&CK mapping, detection ideas, defensive recommendations, and key learning points.

This structure is directly relevant to AI-assisted investigation. Before AI can help an analyst summarize or triage an incident, the workflow needs clear input categories such as attacker behavior, affected assets, likely intent, indicators, detection logic, and recommended response actions.

The project also builds the communication side of threat intelligence. Clear English summaries, concise risk explanations, and structured recommendations are important for incident response because technical findings must be converted into decisions that analysts, managers, and other stakeholders can act on.

## Portfolio Value

For graduate school applications, this project shows that I can read security material, organize attacker behavior, explain operational risk, and connect threat intelligence to incident response. It also provides a written foundation for future AI-assisted analysis tools.

## Next Development Step

The next related portfolio project should move from written briefs to AI-assisted incident triage. It could analyze public incidents, phishing samples, suspicious login events, or short threat reports, then produce:

- Extracted indicators of compromise
- Attack pattern summary
- MITRE ATT&CK mapping
- Initial risk score
- Suggested incident response actions
